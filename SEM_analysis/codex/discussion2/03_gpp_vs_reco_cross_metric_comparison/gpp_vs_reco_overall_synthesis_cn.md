# GPP 与 RECO 恢复机制的综合对比总述

从整套结果看，GPP 与 RECO 在“前置预测”和“过程解释”两个层次上既有共同点，也存在本质差异。共同点在于，两种通量在 `prepeak` 口径下都能够从峰值前辐射、水分和蒸散背景中提取到恢复记忆，在 `recoverywin` 口径下都更容易被恢复期即时环境准确预测。差异则在于，GPP 的机制更容易围绕“重新激活生产过程”组织，而 RECO 的机制更容易围绕“高呼吸状态持续多久”组织。

如果只比较 SHAP 平均 holdout R2，GPP 与 RECO 在 `prepeak` 口径下几乎相当，分别为 `0.423` 和 `0.409`；在 `recoverywin` 口径下则都是显著提高，分别达到 `0.619` 和 `0.665`。这说明无论是生产还是呼吸，恢复期即时环境都比峰值前背景更接近恢复时间本身。但如果把 SEM 一起纳入比较，结果就变得更有层次：GPP 的 `prepeak` SEM 平均 holdout R2 高达 `0.220`，明显高于其 `recoverywin` 的 `0.156`；RECO 则相反，在用户指定的 pruned process SEM 中，`recoverywin` 平均 holdout R2 达到 `0.593`，远高于 `prepeak` 的 `0.055`。这说明 GPP 更适合把峰值前背景写成结构化恢复记忆，而 RECO 更适合把恢复期即时环境写成结构化过程机制。

从主导变量的角度看，GPP 与 RECO 在 `prepeak` 口径下都频繁出现 `SSRD`、`EVA` 与部分 `PRE`，说明峰值前能量和蒸散背景对两种通量都重要。但 GPP 更容易在 SEM 中保留 `SSRD`、`STRD` 与 `EVA` 的稳定路径，RECO 则更容易转向 `TMP` 和 `VPD`。进入 `recoverywin` 后，两者的共同变量都转向 `PRE`、`STRD`、`VPD` 等恢复期环境，但 GPP 更偏向 `TMP`、`SMrz` 这样的“资源与生长季重启”信号，RECO 更偏向 `EVA(sum)`、`VPD` 这样的“高呼吸背景维持”信号。

因此，若从论文叙事上做最终归纳，可以把 GPP 写成“恢复记忆更早形成、恢复速度由资源重建控制”的通量，把 RECO 写成“恢复记忆较弱但恢复期即时过程控制更强”的通量。前者适合强调峰值前背景怎样决定生产过程能否顺利重启，后者适合强调恢复期水热条件怎样决定呼吸偏离基线状态持续多久。这个差异既符合 SHAP 的统计结果，也符合两个碳通量本身的物理生态意义。
