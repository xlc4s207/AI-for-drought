# V8 全 biome 候选机制比较：严格过程版

## 版本定位

这一版是在 V7 的基础上进一步收紧变量与结构，形成“严格过程版”候选比较。它更强调最小必要路径，即尽量用更少的变量和更短的链条来表达恢复时间的过程机制。

## 版本内容

- 仍然是 `24` 个模型。
- 非湿地候选模型变为 `M0_direct`、`M1_water`、`M2_canopy`、`M3_joint`。
- 湿地候选模型变为 `M0_direct`、`M1_water`、`M2_dualwater`、`M3_altwater`。
- 与 V7 相比，这版明显减少了每个模型的特征数、方程数和路径数。

## 主机制筛选结果

| Biome | 主机制模型 | 主要含义 |
|---|---|---|
| Cropland | `M1_water` | 简洁水分链已足以解释主要过程 |
| Forest | `M1_water` | `postpeak30_p_minus_et -> postpeak30_SMrz_mean -> recovery time` 仍是主轴 |
| Grassland | `M1_water` | 两个时间尺度的 `SMrz` 仍为核心 |
| Savanna | `M1_water` | 与 `M0_direct` 拟合几乎并列，说明结构极简化后仍保留水分主轴 |
| Shrubland | `M1_water` | 极简水分机制优于更复杂冠层模型 |
| Wetland | `M1_water` | 在严格口径下，湿地也退化为更简单的水分双节点结构 |

## 与 V7 的核心差异

1. V7 中 Wetland 倾向 `M2_dualwater`，而 V8 中 Wetland 的主机制回到 `M1_water`，说明一旦严格限制结构复杂度，湿地的最优解释会偏向更简洁模型。
2. V7 中很多 biome 仍保留事件属性作为直接路径，而 V8 更强调由恢复窗内部状态变量直接决定目标。
3. 这一版中 `M1_water` 在 6 个 biome 中几乎形成统一的主机制框架，说明“水分路径作为最小充分解释”的一致性更强。

## 结果解读

- Forest：`postpeak30_p_minus_et` 是最强直接负路径，也是最强总效应来源。
- Grassland 与 Cropland：`postpeak30_SMrz_mean` 和 `postpeak60_SMrz_mean` 形成一负一正的成对结构，反映恢复窗早期与后期土壤水状态的不同生态意义。
- Shrubland 与 Savanna：路径结构相似，但系数绝对值整体小于森林和草地。
- Wetland：`postpeak30_SMs_mean` 强负向、`postpeak30_SMrz_mean` 强正向，两者在更简洁框架中仍是主控因子。

## 图件清单与说明

### `primary_model_paths/`

- 含义：严格版中每个 biome 的主机制路径图。
- 用途：适合放在补充材料中，证明即使压缩结构，主结论依然以水分路径为核心。

### `candidate_model_summary.md`

- 含义：严格版 24 个模型的总比较表。
- 用途：查看方程数、路径数、拟合优度和直接预测变量口径。

### `primary_mechanism_summary.md`

- 含义：严格版主机制的原始摘要。
- 用途：核对每个 biome 的最优过程模型与路径系数。

## 推荐定位

- 如果目标是寻找“最简洁、最稳健、最容易在正文中讲清楚”的机制框架，V8 比 V7 更有优势。
- 如果目标是保留更多可能的生态传导链，V7 更丰富。
- 因此，V8 适合作为主文偏方法学保守的候选版本。

## 为什么后续没有直接采用这一版

这一版已经比 `V7` 更简洁，但它仍然属于“先预设机制模板、再比较谁更好”的候选模型框架。后续正式主线希望采用的是“先由 SHAP 稳定筛出重要特征，再做 SEM”的流程，因此没有直接沿用 `V8` 作为最终版本。另外，`V8` 在多数 biome 上都收敛到 `M1_water`，虽然说明水分机制很稳健，但也容易让不同 biome 的生态差异表达得不够充分，所以最终更多被保留为一个方法学上很有价值的过渡版本。
