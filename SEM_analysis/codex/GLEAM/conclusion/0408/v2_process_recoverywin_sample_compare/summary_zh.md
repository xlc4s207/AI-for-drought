# V2 抽样稳定性比较版本：`5000 / 10000 / 20000`

## 版本定位

这一版本的核心问题不是改模型结构，而是回答一个方法学问题：在全局样本数达到 `1,322,953` 的前提下，SHAP 解释阶段只抽 `5000` 行是否足够稳定。

因此，本版本只改变 SHAP 解释抽样规模，不改变 recoverywin 原始特征集合与 SEM 结构。

## 方法设置

- 建模数据行数：`1,322,953`
- 特征数：`17`
- 模型：`LightGBM + TreeSHAP`
- 比较样本量：
  - `5000`
  - `10000`
  - `20000`

## 比较目的

1. 检查不同 SHAP 抽样下全局重要性排序是否稳定。
2. 观察蜂巢图形态是否在样本增加后趋于平滑。
3. 判断后续主分析中应优先采用 `5000`、`10000` 还是 `20000`。

## 主要结论

1. `5000` 已经能较好反映头部重要变量，适合做快速解释。
2. 但随着样本量提升到 `10000` 和 `20000`，蜂巢图分布更平滑，靠前若干变量的相对顺序更稳定。
3. 因此，后续正式对比与稳健性分析更适合采用 `20000` 抽样。
4. 由于这一版本只比较 SHAP 抽样，SEM 路径结构本身没有变化，所以路径图仍参考 V1 的原始路径图。

## 图件清单与说明

### `beeswarm_5000.png`

- 含义：`5000` 抽样下的全局蜂巢图。
- 解读：适合快速看变量影响方向，但离散程度较大。

### `beeswarm_10000.png`

- 含义：`10000` 抽样下的全局蜂巢图。
- 解读：与 `5000` 相比，分布更平滑，头部变量的影响带更清晰。

### `beeswarm_20000.png`

- 含义：`20000` 抽样下的全局蜂巢图。
- 解读：是三档中最稳定、最适合正式展示的一版。

### `importance_topk_5000.png`、`importance_topk_10000.png`、`importance_topk_20000.png`

- 含义：三档样本下的全局重要性条形图。
- 解读：可直接比较样本量变化是否改变头部特征排序。

### `path_overview_reference.png`

- 含义：SEM 参考路径图。
- 说明：因为本版本关注的是 SHAP 抽样稳定性，而非路径结构变化，所以路径图沿用原始版本作为参考。

## 结论建议

如果只是做探索性分析，`5000` 已足够；如果作为正式版本输出，建议采用 `20000`。这也是后续去冗余版本与扩展版本继续使用 `20000` 的原因。
