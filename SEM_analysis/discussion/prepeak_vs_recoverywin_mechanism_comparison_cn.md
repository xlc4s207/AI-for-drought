# GPP 骤旱恢复时间的 `prepeak` 与 `recoverywin` 机制对比分析

本文的目标是比较两套针对 `GPP code1 flash SMrz` 恢复时间的解释框架，并明确它们各自回答的科学问题。`prepeak` 方案使用从 `onset_start` 到 `peak` 的事件前段特征，关注的是“在恢复开始之前，峰值前的水热过程是否已经携带了后续恢复难度的信息”；`recoverywin` 方案使用恢复阶段本身的环境均值特征，关注的是“真正进入恢复窗之后，哪些近端环境过程直接控制恢复速度”。这两个口径并不是简单的替代关系，而是分别对应恢复问题的前置可预报性与恢复期近端控制两个层次。

本稿的讨论建立在已经生成的 `prepeak vs recoverywin` 对比结果和底层 SHAP/SEM 数据之上。直接用于可视化比较的结果来自 `process/SEM_analysis/codex/GLEAM/results/gpp_code1_flash_smrz_compare_prepeak_vs_recoverywin_20260420`，其中包括 5 张 biome 级 dependence 对比图，即 `Forest_prepeak_vs_recoverywin_dependence.png`、`Grassland_prepeak_vs_recoverywin_dependence.png`、`Savanna_prepeak_vs_recoverywin_dependence.png`、`Cropland_prepeak_vs_recoverywin_dependence.png` 与 `Shrubland_prepeak_vs_recoverywin_dependence.png`，以及跨 biome 的总蜂巢图 `all_biomes_prepeak_vs_recoverywin_beeswarm.png` 和元数据汇总表 `summary.csv`。这些图件由 `process/SEM_analysis/codex/GLEAM/code/run_plot_prepeak_vs_recoverywin_comparison_20260420.sh` 调用 `process/SEM_analysis/codex/GLEAM/code/33_plot_prepeak_vs_recoverywin_comparison.py` 生成。因此，本文的结论建立在已经完成的图像比较结果、脚本读取的底层数据文件以及后续汇总统计之上。

底层 SHAP 数据来自两个结果根目录。`prepeak` 方案的数据根目录为 `process/SEM_analysis/codex/GLEAM/results/gpp_code1_flash_smrz_onsetpeak_clean/prepeak_event_shap_sem_20260420/shap_by_biome`；`recoverywin` 方案的数据根目录为 `process/SEM_analysis/codex/GLEAM/results/gpp_code1_flash_smrz_rechunk_py_clean/shap_process_recoverywin_precipEmean_sample50k_by_biome`。在每个 biome 下，本文实际参考了 `feature_importance.csv`、`dependence_plot_data.parquet`、`dependence_sample_features.parquet`、`dependence_sample_shap_values.parquet` 和 `run_summary.txt`，并结合 `gpp_code1_flash_smrz_compare_prepeak_vs_recoverywin_20260420/summary.csv` 记录的作图元数据进行核对。另外，为了把 SHAP 现象与机制路径联系起来，本文还参考了 `prepeak` 的 `process/SEM_analysis/codex/GLEAM/results/gpp_code1_flash_smrz_onsetpeak_clean/prepeak_event_shap_sem_20260420/sem_by_biome/*_sem_summary.txt` 与 `*_estimates.csv`，以及 `recoverywin` 的 `process/SEM_analysis/codex/GLEAM/results/gpp_code1_flash_smrz_rechunk_py_clean/sem_process_recoverywin_precipEmean_usertrim/by_biome/*_sem_summary.txt` 与 `*_estimates.csv`。因此，以下分析并不是仅凭图像肉眼判断，而是综合了图像、SHAP 排序、dependence 曲线数据、beeswarm 总图和 SEM 结果。

为了保证严格可比，本文只比较两套方案共同覆盖的五个 biome，即 Forest、Grassland、Savanna、Cropland 和 Shrubland，并且只保留两套方案共同存在的九个非事件特征：`total_precipitation_mean`、`total_evaporation_mean`、`temperature_2m_mean`、`VPD_mean`、`SMrz_mean`、`lai_total_mean`、`ssrd_mean`、`strd_mean` 和 `wind_speed_mean`。这一限制与对比脚本的设计完全一致，因此本文所总结的差异是“相同特征集合、不同时间窗口”下的差异，而不是由变量池不一致造成的表观差异。

从预测表现上看，`recoverywin` 方案在 SHAP 模型上的 holdout R2 始终高于 `prepeak`。五个 biome 的 `prepeak` SHAP holdout R2 分别为 Forest `0.434`、Grassland `0.439`、Savanna `0.448`、Cropland `0.362` 和 Shrubland `0.432`，平均值为 `0.423`；对应的 `recoverywin` SHAP holdout R2 分别为 `0.604`、`0.653`、`0.643`、`0.483` 和 `0.713`，平均值为 `0.619`。这表明恢复窗口内的环境状态距离目标变量 `t_recover_to_baseline_abs_peak` 更近，因而天然具有更强的统计解释力。与此相对，线性机制框架中的 SEM 结果却呈现相反趋势：`prepeak` 在五个 biome 的平均 holdout R2 为 `0.220`，高于 `recoverywin` 的 `0.156`，而且除 Shrubland 外，其余四个 biome 都是 `prepeak` 更高。这一结果提示，`recoverywin` 更像是“近端诊断器”，`prepeak` 更像是“能够较早暴露恢复记忆的机制预报器”。

从特征排序看，两套方案具有共同骨架，但阶段侧重点显著不同。`prepeak` 的 top 5 特征在不同 biome 中更频繁地出现 `SSRD`、`ET` 和部分辐射相关项，例如 Forest 的前五位为 `ET`、`SSRD`、`STRD`、`VPD` 和 `SMrz`，Savanna 的前五位为 `SSRD`、`ET`、`STRD`、`PRE` 和 `SMrz`。相比之下，`recoverywin` 更频繁地把 `PRE`、`VPD`、`T2m` 和 `STRD` 放到靠前位置，例如 Cropland 的前五位为 `PRE`、`VPD`、`STRD`、`SSRD` 和 `SMrz`，Shrubland 的前五位为 `T2m`、`PRE`、`SSRD`、`STRD` 和 `VPD`。如果取五个 biome 的平均绝对 Spearman 相关系数，`prepeak` 对 `SSRD` 的平均绝对相关达到 `0.808`，而 `recoverywin` 仅为 `0.166`；相反，`recoverywin` 对 `VPD`、`Wind` 与 `STRD` 的平均绝对相关分别达到 `0.744`、`0.754` 与 `0.794`，均明显高于 `prepeak`。这说明 `prepeak` 更强地记录了峰值形成阶段的辐射与蒸散耦合背景，而 `recoverywin` 更强地记录了恢复阶段的大气干旱、湍流交换和热湿背景。

`PRE` 的比较最能体现两套口径的物理差异。在 `recoverywin` 中，`PRE` 在所有 biome 都表现为一致的正向总效应，五个 biome 的分位差均为正，范围大致为 `15.2-23.7` 天，说明恢复阶段更多降水能够稳定缩短恢复时间，这符合最直接的补水逻辑。相反，在 `prepeak` 中，`PRE` 的作用明显更弱，也更依赖 biome 背景：Forest、Grassland、Savanna 和 Shrubland 仍为正，但 Cropland 已经接近零甚至局部可表现为负向。也就是说，峰值前的降水并不只是“越多越快恢复”，它还可能与更高生产力背景、更复杂的事件组织方式或更大的可损失碳汇容量耦合在一起。因此，`prepeak PRE` 更接近“事件形成记忆”的载体，而不是恢复阶段那种单义的直接资源。

`VPD` 的对比同样非常关键。`recoverywin` 中，`VPD` 在所有 biome 都对应稳定的正向 SHAP 关系，分位差约为 `4.3-10.7` 天，这表明恢复阶段较高的大气干旱需求会普遍拉长恢复时间，是一个典型的近端限速因素。`prepeak` 中的 `VPD` 则不具备这种单调一致性：Cropland 与 Forest 为明显正向，Grassland 较弱，Savanna 接近零，Shrubland 甚至转为负向。这意味着峰值前的 `VPD` 不能简单等价于“恢复时的大气胁迫”，它更多反映的是干旱快速推进到峰值的背景气候耦合状态。在 `prepeak` 阶段，高 `VPD` 既可能意味着更快的耗水和更强的蒸散，也可能与高辐射、较高冠层活性同时出现，因此其物理角色比恢复阶段更复杂。

温度变量也显示出清晰的阶段分化。`recoverywin` 中，`T2m` 在五个 biome 的分位差全部为负，而且在 Grassland、Shrubland 等 biome 中达到 `-19` 到 `-20` 天量级，说明恢复阶段较高气温总体不利于恢复尾部的缩短。`prepeak` 中，`T2m` 的负向作用仍然存在，但明显更弱，Forest 甚至接近零或略正。这说明峰值前的温度更像是一个背景状态或事件形成条件，而进入恢复阶段后，温度才真正变成直接控制恢复速度的近端约束。也可以说，`prepeak T2m` 更偏“记忆”，`recoverywin T2m` 更偏“即时控制”。

蒸散发的角色几乎完全体现了两种时间窗口的互补性。`prepeak` 中，`ET` 在五个 biome 的分位差全部为正，Forest 高达 `23.8` 天，Savanna 为 `12.9` 天，这说明峰值前较高蒸散发并不只是耗水信号，它同时表征了系统在事件到达峰值之前仍保持较强地气交换与生态活性的背景，这种背景往往意味着较强的恢复潜力。`recoverywin` 中，`ET` 仍多为正，但其解释明显混合化，Forest 和 Savanna 仍为强正，Grassland 接近中等正值，而部分 biome 中的意义已经不再纯粹。这提示在恢复阶段，蒸散发既可能代表植被重新活跃，也可能代表恢复期仍持续失水，因此它不如 `prepeak` 中那样具有清晰的单一生态含义。

土壤水分 `SMrz` 则揭示了 biome 依赖性极强的“同名异义”现象。Forest 在两套方案中都偏负，说明根区供水越充足，恢复越快，这是一种最直观的供水限制模式。Grassland 在两套方案中则都偏正，特别是 `recoverywin` 的分位差达到约 `5.0` 天，提示这里的高土壤水很可能同时标记了更高生产力背景和更大的受损潜力，而不只是“额外资源”。Savanna 的变化最能体现阶段转换：`prepeak` 中 `SMrz` 为弱正，而 `recoverywin` 中转为接近零或略正偏弱，这说明峰值前的土壤水可能更多反映背景生态系统容量，而恢复阶段的土壤水才逐步回到资源限制的角色。Shrubland 则从 `prepeak` 的强负值转为 `recoverywin` 的弱负或接近零，说明干旱区背景下，峰值前的水分储备对后续恢复记忆的影响可能更强。

短波辐射 `SSRD` 是本次比较中最一致也最醒目的阶段差异。`prepeak` 中，`SSRD` 在五个 biome 的分位差全部为负，Forest、Grassland 和 Savanna 分别约为 `-25.3`、`-21.4` 和 `-23.8` 天，说明峰值前较高短波辐射更像是事件形成的放大器，会通过促进蒸散与耗水加速系统到达更深峰值，从而留下更长的恢复尾部。进入 `recoverywin` 后，这一效应几乎全部被削弱，甚至在多个 biome 中转为弱正或接近零。这说明恢复阶段的短波辐射同时提供光合恢复所需的能量，因而其净效应不再像峰值前那样单向地代表胁迫增强。

与 `SSRD` 相对，`STRD` 在 `recoverywin` 中的负向作用更一致。五个 biome 的 `recoverywin STRD` 分位差全部为负，而且 Forest、Savanna 和 Shrubland 都达到 `-13` 到 `-20` 天量级，说明较高长波辐射对应的暖湿、多云、低 VPD 背景更有利于缩短恢复时间。`prepeak STRD` 仍为负，但强度较弱，且其生态含义更偏背景气团状态而非直接近端限制。因此，从辐射项整体看，`prepeak` 更强调“短波驱动的事件强化”，`recoverywin` 更强调“长波对应的暖湿恢复环境”。

综合这些结果，可以把两套口径放进一个更清楚的机制框架中理解。`prepeak` 所捕获的不是恢复阶段本身的即时约束，而是峰值前水分消耗、辐射放大、温度背景和生态活性共同塑造的“恢复记忆”。这个阶段的信息距离恢复发生尚有时间差，因此统计预测力不如 `recoverywin`，但它更适合讨论可预报性，也更容易在线性机制模型中保留稳定结构。`recoverywin` 则抓住了恢复阶段的近端控制因子，尤其是降水补给、VPD、温度和部分长波辐射背景，因此预测力更高，但机制上更接近事后解释，而不是事前预报。

因此，本文建议在论文写作中不要让 `prepeak` 与 `recoverywin` 去竞争“谁更正确”，而应把它们写成一个分层机制故事。`prepeak` 可以被定义为恢复难度的前置预测框架，用来说明系统在峰值到来之前已经部分写入了后续恢复时间的信息；`recoverywin` 则可以被定义为恢复速度的过程解释框架，用来说明真正进入恢复阶段之后，哪些水热条件在近端决定了恢复尾部的长短。把这两者合并起来，才能得到对 GPP 骤旱恢复时间最完整、也最具论文叙事张力的解释。
