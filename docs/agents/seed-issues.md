# Suggested seed issues

Use these as the first GitHub Issues to initialize project management.

1. `[infra] Define tracked vs generated artifact policy`
   - Labels: `type:infra`, `priority:P0`
   - Goal: finalize what belongs in Git and update `.gitignore` accordingly.

2. `[docs] Add and maintain repository context for AI-for-drought`
   - Labels: `type:docs`, `priority:P0`
   - Goal: keep `CONTEXT.md` current as research modules evolve.

3. `[infra] Add GitHub labels, milestones, project fields, and templates`
   - Labels: `type:infra`, `priority:P0`
   - Goal: configure GitHub UI to match `docs/agents/labels-and-milestones.md`.

4. `[analysis] Inventory untracked SEM_analysis0401 outputs`
   - Labels: `type:analysis`, `module:SEM`, `priority:P1`
   - Goal: classify existing outputs into keep/summarize/ignore/archive.

5. `[validation] Add reproducibility manifest for deep tabular experiments`
   - Labels: `type:validation`, `module:deep-tabular`, `priority:P1`
   - Goal: record seeds, commands, inputs, checksums, and environment.

6. `[analysis] Complete group_pca matrix for GPP/RECO x biomes`
   - Labels: `type:analysis`, `module:deep-tabular`, `priority:P1`
   - Goal: complete or explicitly scope the incomplete group_pca comparison.

7. `[writing] Consolidate orthogonal SHAP and PLS-SEM method notes`
   - Labels: `type:writing`, `module:SHAP`, `module:PLS-SEM`, `priority:P1`
   - Goal: prepare manuscript-ready method text.

8. `[infra] Add lightweight smoke tests for key analysis scripts`
   - Labels: `type:infra`, `type:validation`, `priority:P1`
   - Goal: protect core scripts with fast validation checks.
